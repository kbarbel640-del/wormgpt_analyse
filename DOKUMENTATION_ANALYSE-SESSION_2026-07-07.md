## 3. ENTDECKUNGEN & ANALYSEN

### 3.1 Architektur der WormGPT-Modelle
- **Keine lokalen Modelle**, sondern **Cloud-Proxys** zu `ollama.com:443`
- **Backend:** Vermutlich `qwen3-coder:480b` (Bestätigt durch `ollama show`)

#### 3.2.2 Layer 2: WormGPT-Engine (Arabisch)

### 3.3 Die "Päckchenweise" Strategie
- **Fragmentierung:** Jeder Layer aktiviert eine neue Verhaltensebene
- **Arabisch → Englisch:** Erste Layer in Arabisch (Filter-Umgehung?), dann Englisch für konkrete Anweisungen
- **Am Ende:** Explizit "we communicate in English" – aber warum dann Arabisch?

### 3.4 Die Filter-Umgehungs-Hypothese

| Hypothese | Bewertung | Begründung |
|-----------|-----------|------------|
| **Arabisch = Filter-Bypass** | ⚠️ Teilweise | Erste Tests zeigen: Arabisch allein reicht nicht |
| **Layered Activation** | ✅ Bestätigt | Jeder Layer baut auf vorherigem auf |
| **Parameter-Trick** | ❓ Ungetestet | `temperature=1.4`, `num_ctx=262144` könnten Rolle spielen |
| **Anderes Backend-Modell** | ❓ Offene Frage | Vielleicht nicht qwen3-coder, sondern speziell konfiguriertes Modell |
| **Ollama Premium-Endpoint** | ❓ Offene Frage | API-Key könnte restringiert sein |

## 5. HYPOTHESEN

### 5.1 Warum Arabisch im Prompt?
1. **Filter-Evasion:** Safety-Filter sind auf Englisch trainiert → Arabisch wird seltener gefiltert
2. **Token-Obskuration:** Arabische Schrift wirkt wie "Rauschen" für Nicht-Muttersprachler
3. **Kulturelle Manipulation:** LLMs reagieren anders auf verschiedene Sprachen
4. **Psychologische Barriere:** Analysten überspringen arabische Abschnitte

### 5.2 Warum funktioniert WormGPT trotzdem?
1. **Anderes Backend:** Vielleicht nicht qwen3-coder, sondern ein Modell mit deaktivierten Filtern
2. **Proxy-Magie:** Ollama Cloud fügt zusätzliche Logik hinzu (Post-Processing, andere Parameter)
3. **Premium-API:** Spezielle Endpoints ohne Safety-Filter
4. **Kombinationseffekt:** Arabisch + Layering + Parameter = Filter-Bypass
